Chapter 3
Induction

In the previous chapter we defined the basic notions of set theory, and mostly avoided the topic of infinite sets. In this chapter, we tackle the subject of infinity heads-on. Finite sets are rather boring, we can simply examine them and figure out whether they satisfy any given predicate by manually checking against the definition of that predicate. With infinite sets, this is not possible anymore: checking whether some property is fulfilled by looking at all members of an infinite set would literally take forever. Instead, we need to find clever ways of proving properties; infinite quantities are in fact the reason why we need to bother with proofs, deductive reasoning and mathematics in the first place.

3.1: Infinite Troubles

It turns out that constructing an infinite set from first principles is not a trivial task. We cannot just list all its elements, because we want our mathematical notation itself to be of finite size. Instead, we can use the technique we introduced for set builder notation: we can define a set by giving a predicate that is satisfied by exactly the members of the set.
A simple predicate that is satisfied by infinitely many objects is that of being a set; consider {x∣x is a set}\left \lbrace x \mid \text{$x$ is a \href{http://localhost:1234/sets.html#set}{\htmlClass{ref definition}{\htmlData{preview=http://localhost:1234/previews/set.html}{set}}}} \right \rbrace {x∣x is a set}, the set of all sets. This construct looks innocuous enough, but closer inspection reveals some problematic properties. The set of all sets is a set itself. And as such, it has to contain itself.
A set that contains itself is already unsettling, as it makes the very natural notion of grouping objects together feel a lot less natural. But we can reveal even more problematic inconsistencies by getting slightly more creative with our choice of predicate. Apparently, it should not be possible to define sets via arbitrary predicates. In order to not lose the convenience of set builder notation, we can still allow definitions via predicates, as long as the predicate is only applied to members of some pre-existing set(s). Intersection, union, and set difference are all defined via predicates, but the predicate need only be applied to the elements of the two sets to be combined1

Having narrowly prevented mathematics from self-destructing, we must unfortunately admit that we are still unable to construct sets of infinite size. For that reason, we introduce a new way of defining sets in the next section, a technique that allows us to describe infinite sets without introducing any inconsistencies.

3.2: Structural Induction

The core idea behind safely defining an infinite set is to start out with some non-empty set, and to allow constructing new, larger members of the set from already existing members of the set. Any such newly constructed member can then be used to construct even larger members, ad infinitum.
Consider the following example of a set of blue and pink lamps. We start out with two different kinds of simple lamps; one kind emits blue light, the other kind emits pink light. We also have inverters, which can be placed over any lamp to switch its color. Finally, we can combine any two lamps with a pink-amplifier, the resulting lamp is pink if any of its two sub-lamps is pink, or blue if both sub-lamps are blue. Example 3.1 lets you build such lamps.
Example 3.1: Building Lamps
A hierarchical depiction of a lamp. Hover over any circle to modify the lamp. “⊤\top⊤” denotes a simple blue lamp, “⊥\bot⊥” a simple pink lamp, “¬\neg¬” an inverter, and “∧\land∧” a pink-amplifier.
The color of the topmost circle indicates the color of the lamp as a whole.
This silly example demonstrates several interesting properties. Clearly, there are infinitely many distinct lamps, as you can, for example, create arbitrarily long chains of inverters. But not only have we defined an infinite set of lamps, we have also defined how to assign a color to each of its infinitely many members. And despite there being infinitely many lamps to consider, you probably have a good understanding of some properties they all share; for example, you will agree that every lamp that consists only of simple pink lamps and pink-amplifiers is going to be a pink itself. Take a moment and try to pinpoint exactly why that statement must be true even though you cannot check it for every single possible lamp.
We will now take a detailed look at the traditional mathematical notation for defining infinite sets and working with them.

    Perhaps surprisingly, the definition of lamps does not involve their colors at all. All we have defined so far is a set of certain sequences of symbols according to some hierarchical structure, without any associated meaning. In general, such sets of hierarchy-encoding symbol sequences are called syntax. Syntactic objects have no properties beyond their inherent structure; two such objects are equal if they have identical structure.
    All mathematical reasoning starts out from defining some purely syntactic objects without intrinsic meaning. We can then ascribe meaning to them, as demonstrated in the following section where we introduce the notion of the color of a lamp.
